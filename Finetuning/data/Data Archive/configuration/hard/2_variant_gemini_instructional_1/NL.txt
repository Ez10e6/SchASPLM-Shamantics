### **System Configuration Specification: 3D Rendering Suite**

#### **1. Objective**
Select exactly one component from each of the three defined categories—Central Processing Units (CPUs), Graphics Processing Units (GPUs), and Power Supply Units (PSUs)—to form a valid system configuration that satisfies all operational, compatibility, and fiscal constraints.

#### **3. System Constraints**

**3.1 Power Capacity Constraint**
*   The aggregate wattage of the selected CPU ($W_C$) and GPU ($W_G$) must be at least 100W less than the maximum capacity of the selected PSU ($W_{cap}$).
*   **Equation:** $W_C + W_G \leq W_{cap} - 100$.

**3.2 Cost-Wattage Alignment (Cost Justification)**
*   The component possessing the highest wattage within the configuration must also be the component with the highest unit cost.
*   **Logic:** If $Comp_i = \text{argmax}(W_C, W_G, W_S)$, then $Comp_i$ must also equal $\text{argmax}(P_C, P_G, P_S)$.

**3.3 Thermal and Electrical Compatibility**
*   **Exclusion A:** The CPU "HydraPro" ($C_2$) cannot be paired with the GPU "VegaLite" ($G_3$).
*   **Exclusion B:** The PSU "Eco600" ($S_2$) cannot be paired with the GPU "StormZ" ($G_1$).

**3.4 Minimum Specification Threshold**
*   The final configuration is prohibited from consisting of the least expensive component from every category simultaneously.
*   **Prohibited Set:** {$C_3$ (LiteCore), $G_3$ (VegaLite), $S_2$ (Eco600)}.

**3.5 Price Variance Constraint**
*   The price differential between any two selected components must not exceed $400.
*   **Equation:** $\max(P_C, P_G, P_S) - \min(P_C, P_G, P_S) \leq 400$.

---

#### **4. Selection Requirements**
*   Identify a triplet $\{C_n, G_n, S_n\}$ where $n$ represents the selection from each category.
*   Verify all five constraints (3.1 through 3.5) are satisfied before finalizing the architecture.
